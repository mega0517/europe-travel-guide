# 에어비앤비 숙소 검토 보고서 스킬

추천(위시리스트 저장)된 에어비앤비 숙소가 **실제 여행 조건에 적합한지** 비판적으로 검토하고, 도시별·종합 보고서를 작성한다. 추천을 새로 만드는 것이 아니라(그건 poi-curation), 이미 추천된 것을 **평가·검증**한다.

## 워크플로우

### 1. 검토 범위 확정
- 가이드의 `ROUTE`(index.html 임베드)에서 **숙박 도시와 숙박일(checkin=day, checkout=day+nights)**, **인원(4인=2쌍 부부)**, **계절(여름)**, **차량 여행** 조건을 읽는다.
- 검토 대상 = 각 숙박지의 추천 에어비앤비 숙소(위시리스트). 도시별 위시리스트 id는 index.html의 `AB_WISHLIST` 맵에 있다.

### 2. 데이터 수집
추천 숙소의 실제 속성을 모아야 평가가 가능하다. 두 경로 중 가능한 것을 쓴다.

- **(우선) 에어비앤비 위시리스트 직접 추출** — `cheliped-browser`로 로그인 세션에서 각 위시리스트(`/wishlists/{id}`)를 열어 숙소 room id를 모으고, 각 room 페이지(`/rooms/{id}?check_in&check_out&adults=4`)에서 침실 수·가격·평점·리뷰 수·예약가능 여부를 읽는다.
  - **한국어 계정이면 URL에 `&locale=en`을 붙여** 영어 aria-label/문구(Reserve, Minimum stay 등)로 안정적으로 파싱한다.
  - 예약가능 판정: 룸 페이지에 **Reserve 버튼 + 총액**이 있으면 가능, **"Minimum stay" / "Change dates" / "not available"**이면 불가.
  - rate-limit으로 막히면 1회 쉬었다 재시도, 그래도 막히면 확보분으로 진행하고 미수집 범위를 보고서에 명시.
- **(대안) index.html의 POI 큐레이션 데이터 + 알려진 검증 결과** — 라이브 추출이 불가하면 POI.{city}.airbnb와 직전 검증 기록(예약가능 개수, 적용 필터)으로 정성 평가한다. 이 경우 "라이브 미검증" 한계를 명시한다.

수집 데이터는 `_workspace/NN_airbnb-reviewer_data.json`에 저장한다(감사 추적).

### 3. 평가 — 검토 루브릭
각 숙소를 아래 축으로 평가한다. 도시 단위로 집계해 충족률을 낸다.

| 축 | 기준 | 판정 |
|----|------|------|
| 적합성 | 침실 ≥ 2 (2쌍 독립 침실), 4인 수용, 통숙소(개인실·셰어 제외) | 필수 |
| 차량 | 무료 주차(자체/인근) | 필수 |
| 계절 | 더운 지역(밀라노·피란 등) 에어컨 / 냉장고(주방 보유) | 더운 지역 에어컨 필수, 냉장고 권장 |
| 예약 가능성 | 해당 숙박일에 실제 예약 가능(최소숙박·가용성 통과) | 필수 |
| 가성비 | 가격대(€/€€/€€€), 4인 1박 환산 합리성 | 평가 |
| 위치 | 도심 접근성·주차 편의(차량 취침 중심엔 외곽+주차 OK) | 평가 |
| 평판 | 평점(≥4.7 우수) · 리뷰 수(표본 신뢰도) | 평가 |

**판정 규칙:** 필수 축을 하나라도 못 채우면 "부적합(제외 권고)". 필수 충족 + 가성비/위치/평판 우수 → "추천(Top 픽)". 그 사이는 "보통".

### 4. 보고서 작성 — 구조
최종 보고서는 아래 구조의 Markdown으로 작성한다(기본 파일명 `숙소_검토_보고서.md`).

```
# 추천 에어비앤비 숙소 검토 보고서
- 검토일 / 여행 개요(여정·인원·계절·차량) / 검토 대상(도시·도시당 숙소 수)
- 검토 기준 요약(루브릭 한 문단)

## 종합 요약
- 도시별 충족 현황 표: | 도시 | 숙박일 | 숙소수 | 침실2 | 무료주차 | 에어컨 | 예약가능 | 종합 |
- 핵심 인사이트 3~5개(강점·공통 리스크)

## 도시별 검토 (도시마다)
- 총평(2~3문장)
- 기준 충족 요약 + 가격 분포
- ⭐ Top 3 추천(이름·근거)
- ⚠ 주의/부적합 숙소(있으면, 사유)

## 종합 리스크 & 권고
- 도시 간 비교, 일정상 리스크(예: 다박 도시 가용성), 개선 권고

## 부록(선택)
- 숙소별 상세 표(이름·침실·주차·에어컨·가격·평점·예약가능·링크)
```

작성 원칙:
- **수치와 근거를 보여라.** "좋다"가 아니라 "침실2+주차 충족, 평점 4.8, €€"처럼.
- **리스크를 먼저.** 사용자가 예약 전에 알아야 할 약점을 도시 총평 앞부분에 둔다.
- **간결한 표 + 짧은 산문.** 표로 비교, 산문으로 해석.
- 미수집·미검증 항목은 "—" 또는 "(미검증)"으로 솔직히 표기.

## 참고
- 예약가능 판정·locale=en 파싱·위시리스트 추출의 구체 기법은 프로젝트의 에어비앤비 자동화 이력(CLAUDE.md 변경 이력)과 동일하다.
- 추천 자체를 바꾸려면 이 스킬이 아니라 `poi-curation`을 쓴다(검토≠큐레이션).
