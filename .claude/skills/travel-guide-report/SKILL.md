# 해외여행 가이드 보고서 스킬

여행 데이터를 **한 권의 읽기용 가이드 문서**로 종합한다. 인터랙티브 사이트가 "클릭하며 보는 것"이라면, 이 보고서는 **출발 전 통독하고 인쇄해 들고 다니는 가이드북**이다.

## 워크플로우

### 1. 데이터 수집(단일 출처)
프로젝트 `index.html`의 임베드 상수를 읽어 **여정·POI·숙소·링크**를 모은다(수치 임의 생성 금지).
- `ROUTE`: `trip`(제목·기간·차량), `stops`(도시·국가·좌표·날짜·박수·type), `legs`(구간 거리·시간·통행료·tollNote), `fuel`(연비·유가), `tips`.
- `POI[stopId]`: `hotels`(name·note·price·parking·confirmed·lat/lng), `restaurants`, `highlights`, `airbnb`.
- `AB_WISHLIST` / `AB_WISHLIST_SHARE`: 도시별 위시리스트 직링크·공동편집 링크.
- 있으면 `숙소_검토_보고서.md` / `_workspace/*airbnb-reviewer*` 를 읽어 숙소 종합에 인용.

### 2. 보고서 구성 — 섹션 순서와 내용
여행자 동선(출발→도착) 순서로 작성한다. 기본 파일명 `여행_가이드_보고서.md`.

1. **개요(표지)** — 여행명·기간·인원·차량·경로 한 줄 요약·컨셉. 한눈 표: 출발일/도착일/총 박수/총 운전거리(legs km 합)/예상 통행료(legs toll 합).
2. **여정 타임라인** — 일자별 표: `| 일자 | 이동 구간 | 거리·시간 | 숙박지 | 하이라이트 |`. waypoint(경유)와 stay(숙박) 구분.
3. **운전 · 이동 가이드** — 구간별 거리/시간/통행료/연료비(연비·유가로 환산), 비네트 구매 안내, **주차·진입 제한**(예: 밀라노 ZTL, 구시가 보행자 구역), 터널/카트레인 팁. ROUTE.tips를 녹여 쓴다.
4. **도시별 가이드(숙박지마다)** — 도시 소개(2~3문장) → **가볼 만한 곳**(highlights) → **추천 식당**(restaurants, 가격·특징) → **숙소**(확정 호텔은 강조, 추천 호텔·에어비앤비 + 위시리스트 링크) → **주차·이동 팁**.
5. **숙소 종합** — 도시별 숙소 추천 요약 + (있으면) 숙소 검토 보고서 핵심(예약가능·조건 충족·리스크)을 인용. 위시리스트 공동편집 링크 표.
6. **예산 추정** — 연료비(총 km ÷ 100 × 연비 × 유가), 통행료(legs toll 합), 숙박(박수×추정 단가, "미확정" 허용), 식비/입장료 가이드. 합계와 1인당 환산.
7. **준비물 · 실전 팁** — 운전(국제운전면허·비네트·통행 규칙), 계절(여름 더위·에어컨), 통신·결제·응급. 자가용 2쌍 부부 맥락 반영.
8. **부록** — 링크 모음(가이드 사이트 URL, 도시별 위시리스트 공동편집 링크, 지도), 데이터 출처.

### 3. 작성 원칙
- **표 + 짧은 산문 병행** — 비교·수치는 표로, 해석·동선은 산문으로.
- **결정에 필요한 정보 우선** — 주차/통행료/진입제한/소요시간을 도시·구간 설명 앞부분에.
- **수치는 출처 기반** — 거리·통행료·날짜는 ROUTE에서, 임의값 금지. 모르면 "(미확정)".
- **링크는 실제 값** — AB_WISHLIST_SHARE 등 실제 URL을 넣는다(플레이스홀더 금지).
- **분량은 통독 가능하게** — 도시별 1~1.5쪽 수준. 과한 나열보다 선별.

## 경계(혼동 방지)
- **인터랙티브 지도 사이트**를 만들/고치는 요청 → `leaflet-guide-site`(이 스킬 아님).
- **숙소만 평가/검증**하는 요청 → `airbnb-review-report`(이 스킬은 그 결과를 *인용*만).
- 이 스킬은 **읽기용 종합 문서** 작성에 한정한다.
