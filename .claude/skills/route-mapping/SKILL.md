# 루트 매핑

자가용 여행 여정을 지도에 그릴 수 있는 구조화 데이터로 변환한다.

## 절차
1. 출발지/도착지/박 수/숙박 도시 목록을 확정한다(미정이면 사용자에게 확인).
2. 방문 순서대로 stop 배열을 만든다. 각 stop:
   - `id`(영문 슬러그), `name`(한글), `country`, `lat`, `lng`, `day`(YYYY-MM-DD), `type`(start/stay/waypoint/end), `nights`.
3. 인접 stop 간 `legs`에 대략 거리(km)와 운전시간(h)을 추정해 채운다.
4. 국가별 자가용 팁을 `tips`에 정리: 오스트리아·스위스·슬로베니아 **비네트(고속도로 스티커)** 필수, 이탈리아 도심 **ZTL** 진입 제한, 스위스 일부 산악 구간 자동차 적재 열차 등.
5. `_workspace/01_route.json`으로 저장한다.

## 좌표 기준
도시 중심 좌표를 쓰되, 관광 목적지(융프라우요흐, 블레드 호수 등)는 랜드마크 좌표를 쓴다. 소수점 4자리.

## 데이터 스키마
```json
{ "stops": [{"id":"salzburg","name":"잘츠부르크","country":"오스트리아","lat":47.8095,"lng":13.055,"day":"2026-07-06","type":"stay","nights":1}],
  "legs": [{"from":"budapest","to":"salzburg","km":350,"hours":3.7}],
  "tips": ["..."] }
```
왜: frontend-builder와 poi-curator가 동일 id로 데이터를 조인하므로 id 일관성이 정합성의 핵심이다.
